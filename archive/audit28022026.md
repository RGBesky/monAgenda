### 📌 A — connectivity_plus : Implémentation complète et avancée

**Fichier** : `lib/providers/sync_provider.dart` lignes 308–380

3 providers distincts :

```dart
// 1. Stream brut offline/online
final connectivityStreamProvider = StreamProvider<bool>((ref) {
  return Connectivity()
      .onConnectivityChanged
      .map((results) => results.every((r) => r == ConnectivityResult.none));
});

// 2. État offline combiné avec mode forcé par l'user
final isOfflineProvider = Provider<bool>((ref) {
  final forceOffline = ref.watch(forceOfflineProvider);
  if (forceOffline) return true;
  return ref.watch(connectivityStreamProvider).valueOrNull ?? false;
});

// 3. Auto-sync au retour réseau (avec debounce 5s)
final autoSyncOnConnectivityProvider = Provider<void>((ref) { ... });
```

**Feature bonus non documentée** : `periodicSyncRetryProvider` — fallback **Timer.periodic(60s)** pour Linux où `connectivity_plus` peut être muet, qui dépile les actions en attente même sans événement réseau.

**Mode offline forcé** : `forceOfflineProvider` (`StateNotifierProvider<ForceOfflineNotifier, bool>`) persisté dans `SharedPreferences`. L'utilisateur peut forcer le mode offline manuellement.

**À communiquer à Opus** : Ne jamais recréer ces providers. Utiliser `isOfflineProvider` pour toute logique conditionnelle réseau. Watcher `autoSyncOnConnectivityProvider` au niveau `ProviderScope` dans `app.dart`.
