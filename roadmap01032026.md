### Faisabilité du mode assistant (5.6)

> **Est-ce possible avec Danube 3 500M ?**
> 
> ✅ **Oui, partiellement.** Danube 3 est un SLM (Small Language Model) fine-tuné sur l'extraction d'information structurée. Il peut :
> - Parser des questions simples en intentions (QUERY_NEXT, QUERY_COUNT, QUERY_WHEN)
> - Extraire des entités (personne, date, type d'event) depuis une question naturelle
> - Produire un JSON structuré `{"intent": "QUERY_NEXT", "person": "Marc", "type": "réunion"}`
> 
> ❌ **Ce qu'il ne peut PAS faire :**
> - Générer des phrases naturelles fluides (ce n'est pas GPT-4)
> - Raisonner sur des données complexes (agrégations, comparaisons temporelles)
> - Maintenir un contexte conversationnel multi-tour
> 
> **Stratégie proposée :** Danube génère un JSON d'intention → le code Dart execute la query SQLite → le résultat est formaté en template Dart ("Votre prochain rdv avec Marc est le 5 mars à 14h"). Le LLM n'écrit PAS la réponse, il extrait l'intention.
